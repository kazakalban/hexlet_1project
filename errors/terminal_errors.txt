Эта ошибка возникает, когда вы пытаетесь выполнить установку с флагом --user внутри виртуального окружения. В виртуальных окружениях Python пакеты устанавливаются локально для этого окружения, и флаг --user не имеет смысла, так как он используется для установки в пользовательские каталоги вне виртуального окружения.
Как исправить:

    Если вы работаете внутри виртуального окружения, просто уберите --user:

python3 -m pip install dist/*.whl

Если вы не хотите использовать виртуальное окружение, выйдите из него:

deactivate

Затем повторите команду с --user:

    python3 -m pip install --user dist/*.whl

Рекомендация:

Используйте виртуальные окружения для изолированной установки пакетов, чтобы избежать конфликтов с глобальными или пользовательскими пакетами.


Когда вы переименовываете репозиторий на GitHub, ваш локальный репозиторий всё ещё ссылается на старое имя. Чтобы обновить информацию о репозитории в вашем локальном репозитории, выполните следующие шаги:
1. Проверьте текущую конфигурацию remote:

git remote -v

Это покажет URL старого репозитория (например, https://github.com/username/main.git).
2. Обновите URL remote:

Используйте команду git remote set-url для обновления ссылки на новый репозиторий:

git remote set-url origin https://github.com/username/index.git

3. Проверьте, что всё обновлено:

Снова выполните:

git remote -v

Теперь должен отображаться новый URL (например, https://github.com/username/index.git).
4. Проверьте соединение:

Убедитесь, что соединение работает:

git fetch
