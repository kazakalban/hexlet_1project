вопрос  - Подключите в зависимости библиотеку prompt командой poetry add prompt
ответ - Команда poetry add prompt используется в системе управления зависимостями Python под названием Poetry для добавления новой зависимости — библиотеки prompt — в ваш проект. Вот как это работает:
Что происходит:

    Poetry проверяет имя библиотеки:
        Poetry ищет библиотеку с именем prompt в реестре пакетов Python (обычно это PyPI).

    Добавление зависимости:
        Если библиотека найдена, она добавляется в ваш файл pyproject.toml как зависимость.

    Установка библиотеки:
        После добавления Poetry автоматически загружает и устанавливает библиотеку в виртуальное окружение вашего проекта.
Вопрос - как ипортировать из  другой папки функцию 
Ответ - project/
        ├── brain_games/
        │   ├── __init__.py
        │   ├── cpl.py
        ├── scripts/
        │   ├── brain_games.py
1. Убедитесь, что папка brain_games является модулем.

Для этого в папке brain_games должен быть файл __init__.py. Если его нет, создайте пустой файл с именем __init__.py.
2. Импорт функции из cpl.py.

В файле brain_games.py внутри папки scripts используйте следующий импорт:

from brain_games.cpl import <имя_функции>

Где <имя_функции> — это название функции, которую вы хотите импортировать.

Вопрос -  почему при git push Github не загорится 
Ответ -  Условия для отображения зелёного квадрата:

    Коммит привязан к вашему аккаунту GitHub:
        Убедитесь, что в вашем локальном Git настроен правильный email, который привязан к вашему аккаунту GitHub:

git config --global user.email

Если email отличается, обновите его:

    git config --global user.email "ваш_email@привязан_к_github.com"

Коммит сделан в ветке, которая отображается на GitHub:

    Если вы делаете изменения в частной ветке, они засчитываются только после того, как эта ветка объединится с основной (например, main или master).

Репозиторий не является форком без изменений:

    Если это форк репозитория и вы не вносили никаких изменений, ваши коммиты могут не засчитываться. Чтобы вклад отображался, вносите свои изменения.

Репозиторий не является приватным (если настройки профиля скрывают активности):

    Если ваш репозиторий приватный, активность будет видна только вам, если вы не включили отображение приватной активности в настройках профиля GitHub:
        Перейдите в Settings > Profile Settings > Contributions и включите опцию отображения активности в приватных репозиториях.
